POV keys follow the same mode gate as the native shell: F, B, and N act only while in POV mode, and Tab toggles between map and POV. POV mode itself requires WebGPU — there is no CPU 3D fallback; losing the device returns the viewer to map mode with a warning.

Feature Tiers

WebLow favors small caches and inline execution, WebMid targets ordinary desktop browsers, and WebHigh enables larger visible windows and refinement where supported. Startup benchmarking records generation and frame-pacing signals for diagnostics. Tiers change budgets and presentation only.

Storage

IndexedDB is used when the browser allows it. The app stores sparse vault/session records only, never generated terrain, organisms, or renderer output. Export/import remains file based and server-free.

Browser Requirements

WebGPU is optional. When available, the browser selects the WebGPU atlas presentation path; device loss or unsupported adapters fall back to the CPU-composed map without changing world state. Workers and shared memory can run in inline, worker-pool, or shared-memory worker-pool modes. The inline path remains the deterministic fallback.

POV Mode

POV mode hosts the same shared 3D terrain renderer as the native shell on a WebGPU canvas. Hold the primary button and drag to look; pointer capture keeps that drag active outside the canvas. Move with WASD/arrows (Space/Shift fly up/down), and use the mouse wheel to adjust the fly or walk speed. If no WebGPU adapter or device is available — or the device is lost — the viewer returns to map mode with a warning.

Unsupported

Networking, accounts, hosted worlds, multiplayer, and authoritative GPU simulation are outside Phase 7.
